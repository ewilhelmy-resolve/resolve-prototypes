Data Source Sync Status
========================

Sends data source sync status updates to Rita via RabbitMQ.

PREREQUISITES
-------------
Set these workflow variables before using:
• %rita_rabbitmq_url%
• %rita_data_source_status_queue_name%
• %tenant_id%
• %connection_id%
• %sync_status% (sync_completed | sync_failed)

OPTIONAL VARIABLES
------------------
• %sync_error% - Error details (for sync_failed)
• %documents_processed% - Count of synced documents (for sync_completed)

HOW IT WORKS
------------
All fields are populated from workflow variables, typically set by:
• Previous automation script (e.g., data source connector)
• Conditional logic based on sync result
• Try-catch error handling blocks

FIELDS
------
• Status: One of sync_started, sync_completed, sync_failed
• Sync Error: Error description (when status = sync_failed)
• Documents Processed: Document count (when status = sync_completed)

ERROR HANDLING — CREDENTIAL/PERMISSION FAILURES
------------------------------------------------
When KB sync fails due to credential or permission issues, the platform
MUST send "sync_failed" with a descriptive error_message.

Rita detects credential errors by checking if error_message contains
"authentication" or "permission" (case-insensitive). When detected, the UI
shows a specific "Re-verify credentials" action button to the user.

IMPORTANT: Do NOT send "sync_completed" with documents_processed=0 when
credentials fail. Always send "sync_failed" with an error_message.

Recommended error messages:
• Authentication: "Authentication failed: invalid or expired credentials.
  Please re-verify your connection credentials."
• Permission: "Permission denied: the service account does not have
  sufficient permissions to read the requested resources."

ZERO DOCUMENTS
--------------
If sync completes successfully but finds 0 documents (e.g., empty
knowledge base), send "sync_completed" with documents_processed=0.
Rita will show a warning (not error) to the user.

EXAMPLES
--------
Completed:
- %sync_status%: "sync_completed"
- %documents_processed%: "150"

Completed (zero documents — shows warning):
- %sync_status%: "sync_completed"
- %documents_processed%: "0"

Failed (generic):
- %sync_status%: "sync_failed"
- %sync_error%: "Connection timeout after 30s"

Failed (credential error — shows re-verify button):
- %sync_status%: "sync_failed"
- %sync_error%: "Authentication failed: invalid or expired credentials. Please re-verify your connection credentials."

Failed (permission error — shows re-verify button):
- %sync_status%: "sync_failed"
- %sync_error%: "Permission denied: the service account does not have sufficient permissions to read the requested resources."

RABBITMQ MESSAGE FORMAT
-----------------------
All messages go to the "data_source_status" queue with type "sync".

{
  "type": "sync",
  "connection_id": "<connection-id>",
  "tenant_id": "<tenant-id>",
  "status": "sync_failed",
  "error_message": "Authentication failed: invalid or expired credentials. Please re-verify your connection credentials.",
  "timestamp": "2026-03-09T00:00:00.000Z"
}
