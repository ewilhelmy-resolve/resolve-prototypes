# Figma Components Directory - No Manual Styling Approach

This directory contains components generated from Figma designs using the shadcn design plugin. **Zero manual styling allowed** - all visual styling is handled automatically by the Figma plugin.

## Directory Structure

```
figma/
├── generated/          # Raw components from Figma plugin (NEVER EDIT STYLES)
├── customized/         # Semantic and functionality enhancements only
├── index.ts            # Export barrel for all Figma components
└── README.md           # This file
```

## Usage

### Importing Components

```typescript
// Import all Figma components
import { FigmaButton, CustomHeroSection } from '@/components/figma'

// Import from specific directories
import { FigmaCard } from '@/components/figma/generated'
import { RitaHeroSection } from '@/components/figma/customized'
```

### No-Styling Component Workflow

1. **Generate**: Use Figma plugin to create pixel-perfect components in `generated/`
2. **Accept**: Use generated components as-is (NO style modifications)
3. **Enhance**: Copy to `customized/` ONLY for semantic/accessibility improvements
4. **Export**: Add exports to appropriate `index.ts` files
5. **Use**: Import and use without any style changes

## Strict Guidelines

### For Generated Components (`generated/`)
- ❌ NEVER manually edit CSS/Tailwind classes
- ❌ NEVER modify any visual styling
- ✅ Use exactly as generated by Figma plugin
- ✅ They are replaced when regenerated from Figma

### For Customized Components (`customized/`)
- ❌ NO visual styling modifications allowed
- ✅ Semantic HTML improvements only
- ✅ Add accessibility attributes (ARIA, roles)
- ✅ Add enterprise compliance data attributes
- ✅ Add TypeScript interfaces and type safety
- ✅ Add business logic and event handlers

## Naming Conventions

- **Generated**: Use original Figma layer names (e.g., `FigmaButton`)
- **Customized**: Prefix with `Rita` (e.g., `RitaButton`)
- **Files**: PascalCase matching component name

## Quality Standards

All components must meet:
- ✅ TypeScript strict mode compliance
- ✅ WCAG 2.1 AA accessibility
- ✅ SOC2 security requirements
- ✅ Rita Go architectural patterns

## Examples

### Basic Usage
```tsx
import { FigmaCard } from '@/components/figma'

export function Dashboard() {
  return (
    <FigmaCard className="dashboard-card">
      <h2>Dashboard Content</h2>
    </FigmaCard>
  )
}
```

### With Rita Go Enhancements
```tsx
import { RitaHeroSection } from '@/components/figma'

export function HomePage() {
  return (
    <RitaHeroSection
      data-testid="home-hero"
      aria-label="Rita Go welcome section"
    >
      <h1>Welcome to Rita Go</h1>
    </RitaHeroSection>
  )
}
```
