# Figma Components Directory - shadcn Design Plugin Workflow

This directory contains components generated from Figma designs using **shadcn Design Figma Plugin**. **Zero manual styling allowed** - all visual styling is handled automatically by AI-powered conversion to shadcn/ui components.

## Directory Structure

```
figma/
├── generated/          # Components from shadcn Design Plugin (NEVER EDIT STYLES)
├── customized/         # Semantic and functionality enhancements only
├── index.ts            # Export barrel for all Figma components
└── README.md           # This file
```

## Usage

### Importing Components

```typescript
// Import all Figma components
import { FigmaButton, CustomHeroSection } from '@/components/figma'

// Import from specific directories
import { FigmaCard } from '@/components/figma/generated'
import { RitaHeroSection } from '@/components/figma/customized'
```

### shadcn Design Plugin Workflow

**UX Person (in Figma):**
1. **Design**: Create designs using shadcn/ui Figma Kit
2. **Name Layers**: Use semantic names (`<button>`, `<card>`, `<h2>`, etc.)
3. **Plugin**: Open shadcn Design plugin
4. **Generate**: Select component → Choose Claude 3.5 Sonnet → Generate code
5. **Share**: Copy generated code or CLI command to Frontend Team

**Frontend Team (in RITA Go):**
6. **Place**: Copy generated code to `generated/` directory (NO style modifications)
7. **Enhance**: Copy to `customized/` ONLY for semantic/accessibility improvements
8. **Export**: Add exports to appropriate `index.ts` files
9. **Use**: Import and use without any style changes

## Strict Guidelines

### For Generated Components (`generated/`)
- ❌ NEVER manually edit CSS/Tailwind classes
- ❌ NEVER modify any visual styling
- ✅ Use exactly as generated by shadcn Design Plugin
- ✅ They are replaced when regenerated from Figma

### For Customized Components (`customized/`)
- ❌ NO visual styling modifications allowed
- ✅ Semantic HTML improvements only
- ✅ Add accessibility attributes (ARIA, roles)
- ✅ Add enterprise compliance data attributes
- ✅ Add TypeScript interfaces and type safety
- ✅ Add business logic and event handlers

## Naming Conventions

- **Generated**: Use original Figma layer names (e.g., `FigmaButton`)
- **Customized**: Prefix with `Rita` (e.g., `RitaButton`)
- **Files**: PascalCase matching component name
