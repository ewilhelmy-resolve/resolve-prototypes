# Figma Components Directory - Figma Premium + Code Workflow

This directory contains components generated from Figma designs using **Figma Premium + Code** (company subscription). **Zero manual styling allowed** - all visual styling is handled automatically by Figma's built-in code generation.

## Directory Structure

```
figma/
├── generated/          # Raw components from Figma Premium Code (NEVER EDIT STYLES)
├── customized/         # Semantic and functionality enhancements only
├── index.ts            # Export barrel for all Figma components
└── README.md           # This file
```

## Usage

### Importing Components

```typescript
// Import all Figma components
import { FigmaButton, CustomHeroSection } from '@/components/figma'

// Import from specific directories
import { FigmaCard } from '@/components/figma/generated'
import { RitaHeroSection } from '@/components/figma/customized'
```

### Figma Premium + Code Workflow

1. **Design**: Create designs in company Figma Premium workspace
2. **Inspect**: Use Figma Dev Mode to review design specifications
3. **Generate**: Use Figma's built-in Code feature to export React + Tailwind
4. **Place**: Copy generated code to `generated/` directory (NO style modifications)
5. **Enhance**: Copy to `customized/` ONLY for semantic/accessibility improvements
6. **Export**: Add exports to appropriate `index.ts` files
7. **Use**: Import and use without any style changes

## Strict Guidelines

### For Generated Components (`generated/`)
- ❌ NEVER manually edit CSS/Tailwind classes
- ❌ NEVER modify any visual styling
- ✅ Use exactly as generated by Figma Premium Code
- ✅ They are replaced when regenerated from Figma

### For Customized Components (`customized/`)
- ❌ NO visual styling modifications allowed
- ✅ Semantic HTML improvements only
- ✅ Add accessibility attributes (ARIA, roles)
- ✅ Add enterprise compliance data attributes
- ✅ Add TypeScript interfaces and type safety
- ✅ Add business logic and event handlers

## Naming Conventions

- **Generated**: Use original Figma layer names (e.g., `FigmaButton`)
- **Customized**: Prefix with `Rita` (e.g., `RitaButton`)
- **Files**: PascalCase matching component name

## Quality Standards

All components must meet:
- ✅ TypeScript strict mode compliance
- ✅ WCAG 2.1 AA accessibility
- ✅ SOC2 security requirements
- ✅ Rita Go architectural patterns
